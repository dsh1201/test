背景介紹

總體基因體學（Metagenomics）指直接從環境樣本中提取全部的基因體進行定序與分析，以了解樣本中微生物的組成及功能。它使用霰彈槍式定序法（shotgun sequencing）對於樣本中所有DNA進行定序，直接分析樣本中全部微生物的基因體，包括目前未能培養或新發現的微生物菌種，彌補了僅能建立在可培養微生物之傳統研究法的限制。

Reference-based 分析優勢： 相較於 Assembly-based 方法，本報告採用的 Reference-based 分析具有計算效率高、結果穩定、適合大規模樣本比較等優點，特別適合已知微生物群落的功能與分類研究。

**應用範圍**

  • 環境微生物學研究：可以分析土壤、海洋等各種環境中的微生物，理解其組成與功能
  • 臨床診斷：整合宿主與微生物基因組訊息，用於疾病診斷、藥物研發等
  • 生物技術研究：探索微生物群落中潛在的代謝途徑、酵素等
  • 進化生物學：了解不同環境下微生物基因組的進化適應
  • 生態學：研究微生物與宿主的互相作用，了解微生物在宿主健康中的角色

**分析方法比較**

分析方法 優點 缺點 適用場景
Reference-based 計算效率高、快速獲得結果 受限於現有資料庫 已知微生物群落分析
Assembly-based 可發現新物種/基因 計算資源需求大、雜訊多 新物種發現、基因挖掘

本報告採用 Reference-based analysis 方法進行樣本中微生物的組成研究，使用 MetaPhlAn 和 HUMAnN 進行物種分類和功能分析。

生物資訊分析流程

生物資訊分析流程是以 Reference-based analysis 方式處理 Shotgun 定序數據。此分析流程包括使用 KneadData 去除低品質的序列，並在對宿主基因體進行比對後排除宿主相關序列。透過 MetaPhlAn 比對至資料庫（ChocoPhlAn）後，進行人類或小鼠微生物的物種分類分析，可量化在樣本中代表性物種和菌株的分佈。此外，利用 HUMAnN 進行功能分析，透過 UniRef 和 MetaCyc 等資料庫分別進行功能分析和代謝路徑分析

定序資料品質管控分析

定序後得到的原始序列中可能會存在部分於樣品製備中所留下的轉接子（adapter）序列，或是定序品質較差的序列，這些情況可能會對後續的分析造成不利影響。為了得到較佳的分析結果，需要在進行實際分析之前，對原始的定序序列進行品質過濾和檢查的步驟。數據根據成對序列 (Read 1 與 Read 2) 進行整理。Raw：原始序列之數量。Trimmed：原始序列進行品質管控後之序列數量。Decontaminated：移除宿主序列後之序列數量。

Alpha 生物多樣性（Alpha diversity）

Alpha diversity 是一種衡量單一樣本生態系統內生物多樣性的方法。Alpha diversity 使用物種（Species-level）資料計算。使用指標有 Chao1、Shannon、Simpson、Pielou 等。數值愈低表示樣本內的生物多樣性愈低，反之亦然。

Beta 生物多樣性（Beta diversity）

Beta diversity 用於暸解不同群組/樣本間的微生物組成差異，而多變量統計分析（multivariate statistical analysis）使用物種豐度資料（多變量資料）用以衡量不同群組/樣本間的微生物組成結構的方法。樣品組別以不同顏色區分，橢圓範圍為該組別 95% 信賴區間的範圍。當觀察的樣本組別在二維圖上的距離越近，表示相似程度越高，反之則表示樣品組別相似程度越低。

分析方法 基礎 優點 適用情況

差異統計分析

微生物物種豐度資料通常具有高度的變異性、非常態分佈，並可能包含大量的零值或低豐度值。因此，這類數據適用於非參數統計方法，以便更好地處理稀疏數據和大量零值，進而去檢驗其顯著性水準。

統計檢驗說明： p.adjusted 為經過 Benjamini-Hochberg (FDR) 方法校正後的 p 值，用於控制多重檢驗的偽發現率。顯著性標準：*** p < 0.001, ** p < 0.01, * p < 0.05, ns = 不顯著。

PERMANOVA

群落結構差異 - Permutational Multivariate Analysis of Variance

使用Bray-Curtis相似性矩陣和置換檢驗來評估不同組別間微生物群落的整體組成差異。

比較組別 F-value R² p.adjusted 顯著性

說明：PERMANOVA 使用 Bray-Curtis 相似性矩陣去計算距離矩陣。F 檢驗值表示組間變異與組內變異的比值，較大的 F 值表示組間差異較大；R² 值衡量組間差異相對於總體差異的比值，表示對組間差異的解釋度；p.adjusted 為以 Benjamini-Hochberg 方法進行多重檢驗。

採用Bray-Curtis相似性矩陣，透過對資料進行等級排序來衡量不同組別間的相似性差異。

比較組別 R-value p.adjusted 顯著性
載入ANOSIM數據中...
說明：ANOSIM 使用 Bray-Curtis 相似性矩陣去計算距離矩陣。R 值表示組間組內的差異程度，範圍在 -1 到 +1 之間，接近 +1 表示組間差異較大，0 表示組間組內並沒有差異，而接近 -1 表示組內差異較大；p.adjusted 為以 Benjamini-Hochberg 方法進行多重檢驗。

功能性註釋

使用 HUMAnN 3.7 進行功能分析，透過多個功能資料庫進行基因功能註釋。所有豐度結果以 RPK (Reads Per Kilobase) 表示，目的是為了對基因長度進行標準化，以準確比較不同樣本中的豐度。

RPK 標準化說明： RPK = 比對到該功能的讀取數 ÷ 該功能基因長度 (kb)，此標準化方法消除了基因長度差異對豐度比較的影響，使不同基因功能間的豐度具有可比性。
